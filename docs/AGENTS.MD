# AGENTS.MD

## Purpose

`docs` contains durable documentation for architecture, business intent, and operating guidance.

## Business Role

Documentation is part of the product. It preserves implementation rationale, business semantics, and operational rules so future agents and humans can continue development safely.

## Required Documentation Behavior

- keep architecture docs aligned with actual code structure
- keep business docs aligned with actual feature behavior
- when a feature changes, update docs in the same change when the meaning or operation changed

## Update Triggers

Update this file when:

- documentation structure changes
- ownership of docs changes
- new documentation classes are introduced
